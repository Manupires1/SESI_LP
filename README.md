       **COMANDOS TERMINAL E GIT BASH**
git init: Inicia um repositório Git em uma pasta vazia.
git clone: Baixa um projeto que já existe para o seu computador.
git config --global user.name "Seu Nome":Define quem é você no Git.
git config --global user.email "Seu email":identificar você como o autor das alterações no código.
git status:Ele te diz o que foi modificado e o que ainda não foi salvo.
git add:Prepara todos os arquivos alterados para serem salvos.
git push:Envia seus commits locais para o servidor online
git pull:Traz as novidades do servidor online para o seu computador.
pwd:Mostra o caminho da pasta onde você está agora
ls:Lista todos os arquivos e pastas do diretório atual.
cd [nome-da-pasta]:	Entra em uma pasta específica.
cd ..:	Volta para a pasta anterior.
mkdir [nome]:	Cria uma nova pasta.
touch [arquivo.txt]:Cria um novo arquivo vazio.
rm [arquivo]:Remove/deleta um arquivo.
clear:	Limpa a tela do terminal para tirar a "bagunça".
git commit -m "mensagem":Salva permanentemente as alterações com uma nota explicativa.
mkdir: Cria uma nova pasta (ex: mkdir NovaPasta).
touch: Cria um arquivo vazio (ex: touch notas.txt).
cp (Copy): Copia arquivos ou pastas.
mv (Move): Move ou renomeia arquivos e pastas.
rm (Remove): Apaga um arquivo. Para apagar uma pasta e tudo dentro dela, usa-se rm -rf.
cat: Exibe todo o conteúdo de um arquivo na tela.
head / tail: Mostram apenas o início ou o fim de um arquivo, respectivamente.
grep: Procura por uma palavra ou frase específica dentro de um texto.
nano ou vim: Editores de texto integrados ao terminal. O nano é mais amigável para iniciantes.
top ou htop: Mostra os processos que estão consumindo memória e CPU em tempo real.
df -h: Exibe o espaço livre em disco de forma legível.
ping: Testa a conexão com um site (ex: ping google.com).
sudo (SuperUser Do): Executa comandos com privilégios de administrador. Use com cautela!
git remote add origin [url]: Conecta seu Git local a um servidor remoto.
git push origin [branch]: Envia seus commits locais para o servidor.
git branch: Lista os ramos existentes.
git checkout -b [nome]: Cria um novo ramo e já pula para ele.
git merge [nome]: Une as alterações de um ramo de volta ao principal.
